
### 1. Что такое каталог `/etc` в Linux?
`/etc` — это каталог, содержащий **системные конфигурационные файлы**.
### 3. Какие типы файлов обычно находятся в `/etc`?
- Конфигурационные файлы (`.conf`)
- Каталоги конфигураций сервисов
- Файлы инициализации
- Таблицы (например, `fstab`, `hosts`)
### 4. Можно ли хранить бинарные файлы в `/etc`?
в `/etc` должны быть **только текстовые конфигурационные файлы**.
### 5. Для чего используется файл `/etc/passwd`?
Содержит информацию о пользователях:  
логин, UID, GID, домашний каталог, shell.
### 6. Где хранятся хэши паролей пользователей?
**Ответ:**  
В файле `/etc/shadow` (доступен только root).
### 7. В чём разница между `/etc/passwd` и `/etc/shadow`?
- `/etc/passwd` — общедоступный
- `/etc/shadow` — защищённый, хранит хэши паролей и политику их смены
### 8. Для чего нужен файл `/etc/group`?
**Ответ:**  
Хранит информацию о группах пользователей и их членах.
### 9. Что настраивается в файле `/etc/fstab`?
Точки монтирования файловых систем, параметры монтирования и порядок подключения при загрузке.
### 10. Что произойдёт при ошибке в `/etc/fstab`?
Система может:
- загрузиться в **emergency mode**
- не загрузиться вовсе  
    Поэтому изменения нужно делать аккуратно.
### 11. Для чего нужен файл `/etc/hosts`?
Локальное сопоставление IP-адресов и доменных имён без DNS.
### 13. Что хранится в `/etc/resolv.conf`?
Настройки DNS-серверов и домена поиска.
### 15. Что такое `/etc/systemd/`?
в этом каталоге находятся юниты, сервисы, таймеры, overridesы.
### 17. Для чего используется `/etc/crontab`?
для планирования задач с указанием пользователя.
### 19. Какие права обычно имеют файлы в `/etc`?
- Чтение — для всех
- Запись — только для root  
    Обычно: `644` или строже.
