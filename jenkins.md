### **1. Что такое Jenkins и для чего он используется?**
**Ответ:**  
Jenkins — это **open-source-система для автоматизации CI/CD**. Он позволяет:
- Собирать, тестировать и развертывать код.
- Интегрироваться с Git, Docker, Kubernetes и другими инструментами.
- Запускать задачи по расписанию или при изменении кода.
### **3. Что такое Jenkins ПАЙПЛАЙН?**
Это **кодовая конфигурация** на языке Groovy, которая описывает этапы CI/CD.  
### **4. Чем Declarative Pipeline отличается от Scripted Pipeline?**

| **Declarative**                     | **Scripted**                |
| ----------------------------------- | --------------------------- |
| Простой синтаксис (более строгий).  | Гибкий (полный Groovy-код). |
| Подходит для большинства сценариев. | Для сложной логики.         |

### **5. Что такое джоба?**
Это **набор задач, которые выполняются последовательно**.
### **6. Что такое ВЕБХУК в Jenkins?**
**Это способ взаимодействия веб-приложений. вебхуки позволяют отправлять данные из одного приложения в другое при срабатывании какого-либо триггера.
### **7. Что такое Jenkins АГЕНТ?**
Это **удаленная машина**, которая выполняет задачи Jenkins.
- **Master** — управляет заданиями.
- **Agent** — выполняет их.
### **8. Как добавить секреты (Secrets) в Jenkins?**
1. **Manage Jenkins → Credentials → System → Global Credentials**.
2. Добавить: Логин/пароль, SSH-ключи, Токены.
### **11. Что такое Slack?**
 Slack — это корпоративный мессенджер, который интегрируется с Jenkins для автоматического уведомления о статусе сборок
### **12. Что такое Shared Libraries в Jenkins?**
Это **общий код на Groovy**, который можно переиспользовать в разных пайплайнах.  
### **15. Что такое артефакты?**
- Артефакты — это неизменяемые файлы, которые создаются во время выполнения задач сборки в Jenkins, например, при сборке, тестировании или упаковке кода. После создания артефакты архивируются в контроллере Jenkins, чтобы их можно было сохранить и использовать в дальнейшем.
### **20. Как решить проблему «No space left on device»?**
**Ответ:**
1. Очистить **workspace** с помощью cleanWs()  // Удаляет workspace после сборки
2. И Удалить старые сборки:  **Job → Configure → Discard old builds**.
