#### 3. **Какова основная архитектура Ansible?**
- **Control Node**: Узел, где запускаются команды.
- **Managed Nodes**: Узлы, которыми управляет Ansible.
- **Inventory**: Файл с перечнем управляемых узлов.
- **Modules**: Повторно используемые блоки для выполнения задач.
- **Playbooks**: Сценарии автоматизации на YAML.
#### 4. **Что такое Inventory? Какие его виды бывают?**
Inventory — это список серверов, которыми управляет Ansible. Бывает: **Статический**: Простой файл (ini, yaml) и **Динамический**: Генерируется скриптами или плагинами.
#### 5. **Отличие Role от Playbook?**
- **Role** — это **компонент** для организации задач, handlerов, переменных, файлов и шаблонов. Грубо говоря это единица для повторного использования.
- **Playbook** — это **исполняемый сценарий**, который определяет _какие_ хосты и _какие_ роли/задачи нужно выполнить, в каком порядке и с какими параметрами.
#### 6. **Каковы отличия Ad-Hoc команд от Playbook?**
- **Ad-Hoc команды**: Одноразовые команды, запускаются в командной строке (например, перезагрузка сервера).
- **Playbook**: Постоянные сценарии для выполнения сложных процессов.
#### 7. **Как работает Ansible без агента?**
Ansible использует SSH для связи с узлами. Для выполнения задач он копирует модули на узел и запускает их локально.
#### 10. **Что такое Facts в Ansible?**
Facts — это данные о системе, которые Ansible собирает автоматически, например, IP-адреса, ОС, или количество ядер.
#### 14. **Какие виды циклов поддерживает Ansible?**
- **with_items**: Перебор элементов списка.
- **with_dict**: Перебор словаря.
- **loop**: Универсальная замена с гибкостью.
#### 15. **Что такое handlers?**
Handlers — это задачи, которые выполняются при условии, что другая задача вызвала их с помощью `notify`. Например, перезапуск сервиса после изменения конфигурации.
#### 16. **Как выполнять задачи на ограниченном наборе узлов?*
Используйте флаг `--limit` в командной строке или определите группу хостов в Inventory.
