###  **Active Directory (AD)**

#### **Базовые концепции:**

- Что такое Active Directory?  
    _Ответ:_ Это служба каталогов Microsoft для управления пользователями, компьютерами и другими ресурсами в сети. Основана на протоколе **LDAP** и использует **DNS** для именования.
    
- **Компоненты AD**:
    
    - **Domain Controller (DC)** – сервер, хранящий копию AD.
        
    - **Forest, Domain, OU** – иерархия: лес → домены → организационные подразделения.
        
    - **Schema** – шаблон объектов и атрибутов.
        
    - **Global Catalog** – сокращенная база данных для поиска объектов в лесу.
        
- **FSMO Roles** (роли хозяев операций):
    
    - **Schema Master**, **Domain Naming Master** (уникальны для леса).
        
    - **PDC Emulator**, **RID Master**, **Infrastructure Master** (в каждом домене).
        

#### **Глубокая проработка:**

- **Вопрос:** Как работает репликация между DC?  
    _Ответ:_ Через **Multi-Master Replication** (изменения можно вносить на любом DC, затем они синхронизируются). Используются **USN** (номер последовательности обновлений) и **топология сайтов (Sites and Services)** для управления трафиком.
    
- **Вопрос:** Что такое **Trusts** (доверительные отношения)?  
    _Ответ:_ Односторонние/двусторонние связи между доменами или лесами (например, **Transitive Trust** внутри леса).
