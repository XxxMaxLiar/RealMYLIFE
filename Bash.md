Вот 20 популярных вопросов по **Bash** для собеседования с краткими, но содержательными ответами:
---
### 1. Что такое Bash?

**Bash (Bourne Again Shell)** — это командная оболочка и язык сценариев для Unix/Linux систем. Используется для выполнения команд, автоматизации задач и написания скриптов.

---
### 2. В чем разница между `sh` и `bash`?
`bash` — расширенная версия `sh`.  
Отличия:
- Поддержка массивов
- Расширенная арифметика
- Улучшенные конструкции `[[ ]]`
- Автодополнение и история команд
---
### 3. Что означает строка `#!/bin/bash` в начале скрипта?
Это **shebang** — указывает системе, какой интерпретатор использовать для выполнения файла.

---
### 4. Как сделать скрипт исполняемым?

```bash
chmod +x script.sh
```

---
### 5. В чем разница между `$?`, `$$`, `$!`, `$0`?

- `$?` — код завершения последней команды
- `$$` — PID текущего процесса
- `$!` — PID последнего фонового процесса
- `$0` — имя текущего скрипта
    
---
### 6. Что такое переменные окружения?
Это переменные, доступные текущему процессу и его дочерним процессам.  
Создание:

```bash
export VAR=value
```

---
### 7. В чем разница между `"$var"` и `$var`?
`"$var"` сохраняет пробелы и спецсимволы.  
Без кавычек значение может разбиться по пробелам (word splitting).

---

### 8. Что делает `set -e`?
Завершает выполнение скрипта при ошибке (если команда вернула ненулевой код).

---
### 9. Разница между `>` и `>>`?
- `>` — перезаписывает файл
- `>>` — добавляет в конец файла
---
### 10. Что такое пайп (pipe)?
Передача вывода одной команды во вход другой:
```bash
ls | grep txt
```
---
### 11. Как передаются аргументы в скрипт?
- `$1`, `$2` — позиционные параметры
- `$@` — все аргументы
- `$#` — количество аргументов
---
### 12. Разница между `$@` и `$*`?
- `"$@"` — каждый аргумент отдельно
- `"$*"` — все аргументы как одна строка
---
### 13. Что такое `[[ ]]` и чем отличается от `[ ]`?
`[[ ]]` — расширенный тест в bash:
- Поддержка regex
- Нет проблем с word splitting
- Более безопасный синтаксис
---
### 14. Как объявить массив?

```bash
arr=(one two three)
echo ${arr[0]}
```
---
### 15. Как организовать цикл в Bash?
**for:**
```bash
for i in {1..5}; do
  echo $i
done
```
**while:**
```bash
while read line; do
  echo $line
done < file.txt
```
---
### 16. Что такое подстановка команд?
Выполнение команды и подстановка результата:
```bash
result=$(date)
```
---
### 17. Что делает `trap`?
Позволяет перехватывать сигналы:
```bash
trap "echo Exit; exit" SIGINT
```
---
### 18. Что такое дескрипторы файлов?
- `0` — stdin
- `1` — stdout
- `2` — stderr
Перенаправление stderr:
```bash
command 2> error.log
```
---
### 19. Как отладить Bash-скрипт?
- `bash -x script.sh`
- `set -x` внутри скрипта
---
### 20. Как проверить существование файла?
```bash
if [[ -f file.txt ]]; then
  echo "File exists"
fi
```
