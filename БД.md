#### **2. Какие типы БД вы знаете?**
- Реляционные (SQL: MySQL, PostgreSQL, Oracle)
- Нереляционные (NoSQL: MongoDB, Redis, Cassandra)
- Графовые (Neo4j)
- Ключ-значение (Redis, DynamoDB)
- Документные (MongoDB)
- Колоночные (ClickHouse, Cassandra)
#### **4. Чем отличается PRIMARY KEY от UNIQUE?**
- `PRIMARY KEY` — уникальный идентификатор строки (не может быть `NULL`).
- `UNIQUE` — гарантирует уникальность, но может содержать `NULL` (если не `NOT NULL`).
#### **5. Что такое JOIN? Какие виды JOIN бывают?**
`JOIN` объединяет данные из двух таблиц. Виды:
- `INNER JOIN` — только совпадающие строки.
- `LEFT JOIN` — все строки из левой таблицы + совпадения справа.
- `RIGHT JOIN` — все строки из правой таблицы + совпадения слева.
- `FULL JOIN` — все строки из обеих таблиц.
- `CROSS JOIN` — декартово произведение (все комбинации).
#### **6. Что такое индекс? Зачем он нужен?**
Индекс — структура для ускорения поиска (как оглавление в книге).  
индекс создаёт «указатели» на данные, чтобы СУБД не сканировала всю таблицу, а прыгала сразу к нужным строкам.
**Плюсы:** ускоряет `SELECT`, `WHERE`, `JOIN`.  
**Минусы:** замедляет `INSERT`, `UPDATE`, `DELETE` (т.к. индексы надо перестраивать).
**Как работает индекс:**
1. Это отдельная структура данных (обычно **B-дерево**), которая хранит:
    - **Ключ** (значение колонки, по которой строится индекс, например, `user_id`)
    - **Указатель** на физическое расположение строки в таблице (как номер полки и ящика в библиотеке).
2. Когда ты пишешь запрос с `WHERE user_id = 123`, СУБД:
    - Быстро находит `123` в индексе (как поиск по алфавитному каталогу),
    - Получает точный адрес строки,
    - Забирает данные **за 1-2 шага** вместо перебора всей таблицы.
#### **7. Что такое транзакция? (ACID)**
Транзакция — последовательность операций, выполняемых как единое целое.  
**Свойства (ACID):**
- **Атомарность** (всё или ничего).
- **Согласованность** (данные остаются валидными).
- **Изолированность** (параллельные транзакции не мешают).
- **Долговечность** (после фиксации — данные сохранены).
#### **8. Что такое нормализация БД?**
Нормализация — устранение избыточности данных через разбиение таблиц.  
**Основные формы:**
- 1NF (атомарные значения, нет повторяющихся групп).
- 2NF (нет частичных зависимостей от составного ключа).
- 3NF (нет транзитивных зависимостей).
#### **9. Что такое денормализация?**
Намеренное нарушение нормализации для ускорения запросов (например, дублирование данных).
#### **10. Чем отличается DELETE от TRUNCATE?**
- `DELETE` — удаляет строки по условию (можно откатить, медленнее).
- `TRUNCATE` — удаляет все строки (быстрее, нельзя откатить, сбрасывает автоинкремент).
#### **11. Что такое VIEW?**
`VIEW` — виртуальная таблица, результат SQL-запроса (упрощает сложные запросы).
#### **12. Что такое хранимая процедура (stored procedure)?**
Это предкомпилированный SQL-код, хранящийся в БД (ускоряет выполнение, снижает нагрузку на сеть).
#### **13. Что такое триггер?**
Триггер — автоматическое действие (SQL-код), выполняемое при событии.
#### **14. Что такое репликация БД?**
Репликация — копирование данных на несколько серверов для отказоустойчивости и распределения нагрузки.
#### **15. Что такое шардинг (партиционирование)?**
Разделение данных на части (например, по диапазону дат или хэшу) для распределения нагрузки.
#### **16. Как работает CAP-теорема?**
Система может гарантировать только 2 из 3 свойств:
- **Consistency** (согласованность),
- **Availability** (доступность),
- **Partition tolerance** (устойчивость к разделению).
#### **17. Что такое EXPLAIN в SQL?** 
`EXPLAIN` показывает план выполнения запроса (какие индексы используются, где узкие места).
#### **18. Как оптимизировать медленный запрос?**
- Добавить индексы.
- Переписать запрос (избегать `SELECT *`, `LIKE '%text%'`).
- Разбить сложный запрос на части.
- Использовать кэширование.
#### **19. Чем MongoDB отличается от PostgreSQL?**
- **MongoDB** — документная NoSQL-БД (гибкая схема, JSON-документы).
- **PostgreSQL** — реляционная SQL-БД (строгая схема, поддержка JSON).
#### **20. Что такое ORM?**
ORM (Object-Relational Mapping) — технология для работы с БД через объекты в коде (например, Django ORM, SQLAlchemy).
#### **21. Виды бэкапов**
Бэкап - Резервное копирование SQL-баз данных
#####  **Полный бэкап (Full Backup)**
Резервная копия **всей базы данных** на определённый момент времени.
##### **2. Инкрементный бэкап (Incremental Backup)**
Копируются **только изменения с момента последнего бэкапа**.
##### **3. Дифференциальный бэкап (Differential Backup)**
Копируются **все изменения с момента последнего полного бэкапа**.
##### **5. Горячий vs. Холодный бэкап**

| Тип        | Горячий (Hot)                          | Холодный (Cold)                     |
| ---------- | -------------------------------------- | ----------------------------------- |
| **Доступ** | БД работает во время бэкапа.           | БД останавливается.                 |
| **Плюсы**  | Нет downtime.                          | Проще, гарантированная целостность. |
| **Минусы** | Риск несогласованности (если без WAL). | Простой сервера.                    |
