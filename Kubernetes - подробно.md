
## 1. **Что такое Kubernetes и для чего он используется?**
Kubernetes — это система оркестрации контейнеров, созданная для автоматизации развертывания, масштабирования и управления контейнеризированными приложениями. 
## 2. **Что такое Pod?**
Pod — это основная вычислительная единица Kubernetes. Он представляет собой абстракцию, которая:
- Содержит один или несколько контейнеров, работающих вместе как единое целое.
- Делит между контейнерами IP-адрес, хранилище и спецификации (например, лимиты ресурсов).
## 3. **Чем отличается ReplicaSet от Deployment?**
- **ReplicaSet** отвечает за поддержание заданного количества Pod'ов, обеспечивая их доступность. Например, если один Pod выходит из строя, ReplicaSet автоматически создает новый.
- **Deployment** предоставляет более высокоуровневый механизм управления ReplicaSet, включая:
  - Пошаговое обновление Pod'ов для минимизации простоев.
  - Возможность отката к предыдущей версии.
  - Управление версионной историей.
## 4. **Что такое Service?**
Service — это абстракция для организации устойчивого доступа к набору Pod'ов. Он обеспечивает:
- Балансировку нагрузки между Pod'ами, связанными с Service.
- Постоянный DNS-адрес, который не меняется при изменении состояния Pod'ов.
Существует несколько типов Service:
- **ClusterIP**: доступ внутри кластера.
- **NodePort**: доступ извне через фиксированный порт на каждом узле.
- **LoadBalancer**: автоматическое создание внешнего балансировщика (в облачных средах).
## 5. **Что такое ConfigMap и Secret?**
- **ConfigMap**: хранит неконфиденциальные конфигурационные данные, такие как файлы конфигурации, переменные окружения, параметры команд.
- **Secret**: используется для хранения конфиденциальных данных (пароли, ключи API) в зашифрованном виде.
### Различия:
- ConfigMap хранит данные в открытом виде, а Secret — в зашифрованном.
- Secret поддерживает интеграцию с инструментами управления секретами, такими как HashiCorp Vault.

 Горизонтальное масштабирование (Horizontal Pod Autoscaler)
HPA автоматически изменяет количество Pod'ов на основе метрик, таких как загрузка CPU или память. Оно конфигурируется через манифест, задавая минимальное и максимальное количество реплик.

Вертикальное масштабирование
Изменяет выделяемые ресурсы для существующих Pod'ов.
## 8. **Что такое Namespace и зачем он нужен?**
Namespace — это способ разделения ресурсов кластера Kubernetes. Он полезен для:
- Изоляции приложений или сред (например, dev, staging, prod).
- Управления доступом через роли (RBAC).
- Разделения квот на ресурсы (CPU, память).
## 9. **Как работает механизм Service Discovery в Kubernetes?**
Kubernetes автоматически создает механизмы для обнаружения:
1. **Переменные окружения**: при создании Pod'а Kubernetes добавляет переменные окружения с информацией о Service (имя, IP, порт).
2. **DNS**: встроенный DNS-сервер предоставляет DNS-записи для каждого Service. Например, Service с именем `my-service` будет доступен как `my-service.default.svc.cluster.local`.
### Использование:
Pod может обращаться к Service по DNS-имени, и Kubernetes автоматически маршрутизирует запросы к правильным Pod'ам.
