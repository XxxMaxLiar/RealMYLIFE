#### 1. **Что такое Nginx?**
**Nginx** — это **веб-сервер** и **обратный прокси-сервер**, который используется для создания **обратного прокси-сервера**(`proxy_pass`), **обработки HTTP-запросов** 
- **балансировки нагрузки** 
	- `round-robin` (по умолчанию).
	-  `least_conn` (наименьшее количество соединений).
    - `ip_hash` (фиксация клиента за сервером).
- **кэширования контента* (`proxy_cache`)
#### 2. **Чем Nginx отличается от Apache?**
- **Архитектура**: Nginx использует асинхронный событийный подход, тогда как Apache работает с потоками/процессами.
- **Производительность**: Nginx лучше справляется с большим числом одновременных соединений.
- **Конфигурация**: Nginx проще настраивать для задач прокси и балансировки.
#### 3. **Что такое обратный прокси-сервер?**
Обратный прокси принимает запросы от клиентов, пересылает их внутренним серверам и возвращает клиентам ответ. 
#### 4. Чем рестарт отличается от reload?
**отличаются тем, что restart перезапускает сервер, а reload перезагружает конфигурацию**.
restart:
 Используется для значительных изменений в конфигурации, например, изменения портов или интерфейсов.
#### 4. **Какие режимы балансировки нагрузки поддерживает Nginx?**
- **Round Robin** (по умолчанию) — запросы равномерно распределяются.
- **Least Connections** — запрос идет на сервер с меньшим количеством активных соединений.
- **IP Hash** — привязка клиента к определенному серверу по IP.
#### 5. **Что такое файл конфигурации Nginx?**
Это файл, в котором хранятся настройки сервера. Основной файл — `/nginx.conf`. 
#### 6. **у вас есть на локалке поднятый Nginx и он слушает 8081 порт, что это значит?**
«Это продакшен или тестовая среда?»
- **Nginx** настроен принимать **HTTP-запросы** на порту **8081** локальной машины. **Порт** выбран, чтобы избежать конфликтов с другими сервисами (например, если **80 или 443** уже заняты).»
#### 7. **Что такое upstream в Nginx?**
**upstream** — это блок, в котором определяются серверы для прокси или балансировки нагрузки.
#### 8. **Что такое директивы и контексты в Nginx?**
- **Директивы**: команды в конфигурации (например, `listen`, `server_name`).
- **Контексты**: блоки, в которых применяются директивы (например, `http`, `server`, `location`).
#### 9. **Что такое Nginx-логи? Какие типы существуют?**
- **Логи Доступа (access.log)**: информация о каждом запросе.
- **Логи Ошибок (error.log)**: информация об ошибках сервера. 
#### 10. Можно ли установить **https** соединение на порту **8080** или просто **80**?
«**HTTPS нельзя** использовать на **портах 80/8080** по стандарту. Для защищённого соединения нужен порт **443** (или например 8443). Но можно:
1. Настроить **редирект с HTTP (порт 80) на HTTPS (порт 443)**.
2. Поднять **HTTPS** на **альтернативном порту** (например, 8443).
Но будет происходить нарушение стандартов и такое соединение может блокироваться фаерволлами.
#### 11. Объясните, пожалуйста, в чем проблема C10K?
Проблема C10K относится к сетевым сокетам, которые не могут одновременно обрабатывать большое количество клиентов (10 000).
#### Объясните, как запустить Nginx через другой порт, кроме 80?
Чтобы запустить Nginx через другой порт, вам нужно перейти в /etc/Nginx/sites-enabled/ и, если это файл по умолчанию, вам нужно открыть файл с именем «default». Отредактируйте файл и укажите нужный порт.
