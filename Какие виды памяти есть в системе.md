- **RSS (Resident Set Size):** Это часть памяти, которая находится в RAM и не вытеснена в swap. **Это самый важный показатель для нас**, так как он показывает реальное давление на RAM.
    
- **Cache Memory:** Память, используемая ядром для кеширования дисковых операций (например, чтение файлов).
    
- **Virtual Memory (VSZ):** Вся виртуальная память, которую запросил процесс. Она включает RSS, swap, разделяемые библиотеки и другие отображения. Часто бывает сильно завышена.
- «Если копнуть ещё глубже, на уровень Linux, то есть понятия:

- **Page Cache:** Кеш файловой системы, та самая "Cache Memory" в `top`.
    
- **Slab Memory:** Память, используемая ядром для внутренних структур (кэш inode, сетевые буферы). Её можно увидеть командой `slabtop`.
    
- **HugePages:** Специальные большие страницы памяти, которые используются для высокопроизводительных задач (например, БД) чтобы снизить нагрузку на TLB кэш процессора.»
«В работе с Kubernetes мы в основном оперируем понятиями `requests`, `limits` и смотрим на `RSS`-потребление в метриках, чтобы понять, не упирается ли приложение в лимиты.»
