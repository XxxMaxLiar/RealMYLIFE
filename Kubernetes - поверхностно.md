- **Кластер** — группа серверов (нод), работающих как единая система.
    
- **Оркестратор** — инструмент для управления кластером (Kubernetes, Docker Swarm, Nomad).
## 1. **Что такое Kubernetes и для чего он используется?**
Kubernetes — это система оркестрации контейнеров, которая автоматизирует развертывание, управление и масштабирование контейнеризированных приложений. 
Основная цель — упростить управление сложными приложениями, работающими в контейнерах.
### **Ключевые термины для собеседования**

| Термин                | Зачем нужен?                                                         |
| --------------------- | -------------------------------------------------------------------- |
| **Pod**               | Минимальная единица развертывания (1+ контейнеров).                  |
| **Deployment**        | Управляет жизненным циклом подов (обновления, откаты).               |
| **StatefulSet**       | Для stateful-приложений (БД), где важен порядок и постоянство.       |
| **Ingress**           | Маршрутизация HTTP/HTTPS-трафика в кластер.                          |
| **Persistent Volume** | Хранение данных вне контейнеров (чтобы не терялись при перезапуске). |

---

## 2. **Что такое Pod?**
Pod — это минимальная единица развертывания в Kubernetes, представляющая собой один или несколько контейнеров, которые совместно используют сеть, хранилище и спецификации. Обычно Pod включает один контейнер, но может содержать несколько контейнеров для обеспечения тесной кооперации.

---

## 3. **Чем отличается ReplicaSet от Deployment?**
- **ReplicaSet** отвечает за поддержание заданного числа реплик Pods.
- **Deployment** используется для управления обновлениями ReplicaSet и предоставляет более высокий уровень абстракции, включая откаты, контроль версий и возможность пошагового обновления.

---

## 4. **Что такое Service?**
Service предоставляет устойчивую точку входа для доступа к набору Pods, работающих под одним логическим именем. Используется для балансировки нагрузки и маршрутизации запросов к динамически изменяющимся Pod'ам.

---

## 5. **Что такое ConfigMap и Secret?**
- **ConfigMap**: хранит конфигурационные данные в виде пар ключ-значение, не содержащих чувствительную информацию.
- **Secret**: используется для хранения чувствительных данных (пароли, токены) в зашифрованном формате.

---

## 6. **Как работает механизм масштабирования в Kubernetes?**
Масштабирование бывает двух типов:
- **Горизонтальное (Horizontal Pod Autoscaler, HPA)**: изменяет количество Pod'ов на основе нагрузки (например, CPU или памяти).
- **Вертикальное**: изменяет ресурсы (CPU, память) для существующих Pod'ов.

---

## 7. **Что такое PersistentVolume и PersistentVolumeClaim?**
- **PersistentVolume (PV)**: объект, представляющий хранилище, предоставляемое кластером.
- **PersistentVolumeClaim (PVC)**: запрос на использование PV. Связывает Pod с хранилищем через PVC.

---

## 8. **Что такое Namespace и зачем он нужен?**
Namespace — это виртуальное разделение кластера Kubernetes, позволяющее изолировать ресурсы (Pods, Services и т.д.) друг от друга.

---

## 9. **Как Kubernetes управляет состоянием приложений?**
Kubernetes поддерживает желаемое состояние приложения, указанное в манифестах YAML/JSON. Если текущее состояние отличается от желаемого, контроллеры (например, DeploymentController) предпринимают действия для их синхронизации.

---

## 10. **Как работает механизм Service Discovery в Kubernetes?**
Kubernetes использует два подхода:
- **Environment Variables**: при создании Pod'а переменные окружения устанавливаются для подключаемых Service.
- **DNS**: встроенный DNS-сервер автоматически создает DNS-записи для Service, позволяя Pod'ам обращаться к ним по имени.

## 1. **что значит кластеризация контейнеров?**
**Кластеризация контейнеров** — это объединение множества контейнеров в единый (кластер) для повышения отказоустойчивости, масштабируемости и эффективности использования ресурсов.
