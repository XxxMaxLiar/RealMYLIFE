**CI/CD** — это процесс автоматизации, охватывающий этапы разработки, тестирования, развертывания, мониторинга и управления инфраструктурой.
## **1. Разработка (Development)**
**Технологии:**
- **Python** → Основной язык для бэкенда/скриптов.
- **Git** → Контроль версий кода
- **Bash** → Автоматизация задач на серверах.
**Как всё связано:**
1. Разработчик пишет код на Python, фиксирует изменения в Git (репозиторий GitLab).
2. Bash-скрипты помогают в локальной сборке/тестировании.
---
## **2. Непрерывная интеграция (CI)**

**Технологии:**
- **GitLab CI** (или **Jenkins**) → Оркестрация пайплайнов.
- **Docker** → Упаковка приложения в контейнеры.
- **Terraform** → Описание инфраструктуры как кода (IaC).

**Плавный переход:**
1. При пуше в GitLab CI запускает пайплайн:
    yaml
2. **Docker** создает образ приложения (например`Dockerfile` с Python-зависимостями).
3. **Terraform** подготавливает инфраструктуру в **AWS/Yandex Cloud**.    
---
## **3. Тестирование (Testing)**
**Технологии:**
- **PostgreSQL** → Тесты БД.
- **NoSQL/Redis** → Кэширование и тесты NoSQL-запросов.
- **Kafka/RabbitMQ** → Тестирование асинхронной обработки событий.
**Как всё связано:**
1. В CI-пайплайне запускаются юнит-тесты (Python + pytest).
2. Для интеграционных тестов разворачивается временная БД (**PostgreSQL**) и кэш (**Redis**).
3. **Kafka** имитирует потоковую обработку данных.
---
## **4. Непрерывная доставка (CD)**
**Технологии:**
- **Kubernetes** → Оркестрация контейнеров.
- **Ansible** → Конфигурация серверов.
- **Nginx** → Балансировка нагрузки и проксирование.
**Плавный переход:**
1. **Docker-образ** из CI загружается в реестр (GitLab Registry).
2. **Terraform** разворачивает кластер **Kubernetes** в облаке (**AWS/Yandex Cloud**).
3. **Ansible** настраивает виртуальные машины (если не pure Kubernetes).
4. **Nginx** работает как Ingress-контроллер в Kubernetes.
---
## **5. Мониторинг и логирование (Observability)**
**Технологии:**
- **Prometheus** → Сбор метрик.
- **Grafana** → Визуализация метрик.
- **ELK/Kibana** → Анализ логов.
**Как всё связано:**
1. **Prometheus** собирает метрики с **Kubernetes** и **Nginx**.
2. **Grafana** отображает дашборды (например, нагрузку CPU).
3. **ELK** (Elasticsearch + Logstash + Kibana) агрегирует логи приложений.
---
## **6. Обслуживание (DevOps & Admin)**
**Технологии:**
- **Linux-администрирование** → Ручные правки на серверах.
- **ClickHouse** → Аналитика и хранение метрик.
- **RabbitMQ** → Управление очередями задач.
**Плавный переход:**
1. **Администрирование** серверов (**Linux**) требуется для тонкой настройки.
2. **ClickHouse** хранит исторические данные из **Prometheus**.
3. **RabbitMQ** обеспечивает работу фоновых задач (например, обработку PDF).
### **Ключевые переходы:**
1. **От кода к образу**:
    - Python → Docker → Kubernetes.   
2. **От инфраструктуры к мониторингу**:
    - Terraform → AWS → Prometheus → Grafana.
3. **От логов к аналитике**:
    - Nginx → ELK → ClickHouse.
---
### **Что важно запомнить:**
- **GitLab CI/Jenkins** — сердце CI/CD.
- **Docker + Kubernetes** — основа доставки.
- **Prometheus + Grafana** — глаза системы.
- **Terraform + Ansible** — руки DevOps.
