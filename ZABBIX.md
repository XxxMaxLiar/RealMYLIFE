## 1. Из каких компонентов состоит Zabbix?
- **Zabbix Server** — центральный компонент, собирает данные, обрабатывает триггеры, выполняет эскалации.
- **Zabbix Agent** — устанавливается на хост, собирает метрики.
- **Zabbix Proxy** — промежуточный узел для распределённого мониторинга.
- **Frontend (Web-интерфейс)** — PHP-приложение для управления (обычно на Apache/Nginx).

## 2. Чем отличается passive agent от active agent?
### Passive agent
- Сервер **сам запрашивает** данные.
- Агент слушает порт 10050.
- Подходит для серверов с открытым доступом.
- Проще для отладки.

### Active agent
- Агент **сам отправляет данные** на сервер.
- Использует порт 10051.
- Лучше работает за NAT / firewall.
- Меньше нагрузка на сервер (нет постоянных опросов).
Ключевое отличие — кто инициирует соединение.

## 3. Что такое item, trigger, host, template?
### Host
Устройство или сервис, который мы мониторим (сервер, свитч, VM).
### Item
Конкретная метрика.  
Пример: `system.cpu.load`, `vfs.fs.size[/,free]`.
### Trigger
Логическое выражение, определяющее проблему.  
Пример: `avg(/server/system.cpu.load,5m)>5`.
### Template
Набор item'ов, триггеров, графиков и LLD, который можно массово применять к хостам.

## 4. Что такое LLD (Low-Level Discovery)?
LLD — механизм автоматического обнаружения сущностей.
Примеры:
- Автообнаружение файловых систем
- Сетевых интерфейсов
- Дисков
- Docker-контейнеров

LLD правило возвращает JSON, на основе которого создаются item'ы и trigger'ы по шаблону.

## 5. Как работает preprocessing?
Preprocessing — это обработка данных до записи в БД.

## 6. Что такое dependent item?
Dependent item — элемент данных, который зависит от другого item'а (master item).
Используется, когда:
- Есть один большой JSON
- Не хочется делать 10 запросов

Сначала собираем один master item, затем создаём несколько dependent item с preprocessing.
Плюс:
- Снижение нагрузки
- Нет лишних запросов к агенту/API

## 7. Как настроить мониторинг нового сервиса?
Алгоритм:
1. Понять, как получить метрику:
    - Agent
    - SNMP
    - HTTP
    - Script
    - Zabbix sender
2. Создать item.
3. Проверить получение данных.
4. настроить trigger.
5. Настроить action.
6. Добавить графики.
7. При необходимости — вынести в template.
---
## 8. Как работает proxy?
Proxy:
- Собирает данные с хостов
- Буферизует их
- Передаёт на сервер

Режимы:
- Active proxy (сам отправляет данные)
- Passive proxy (сервер опрашивает)
---
## 9. Как уменьшить нагрузку на Zabbix server?
Основные методы:
- Использовать proxy
- Увеличить интервал опроса
- Использовать active agents
- Использовать dependent items
- Использовать preprocessing (discard unchanged)
- Оптимизировать БД (partitioning, housekeeping)
- Отключить ненужные item'ы
- Настроить правильные history/trends
---
## 10. Что такое escalation?

Escalation — это механизм поэтапной реакции на проблему.
Пример:
- 5 минут — письмо админу
- 15 минут — сообщение в Telegram
- 30 минут — звонок
Настраивается через Action → Operations → Escalation steps.

## 11. Какие есть типы триггеров (функции)?
Основные функции:
- `last()` — последнее значение
- `avg()` — среднее
- `min()` / `max()`
- `nodata()` — нет данных
- `count()` — количество совпадений
- `change()` — изменение
- `delta()` — разница
- `strlen()`
- `regexp()`
- `iregexp()`
- `forecast()`
- `trendavg()`

## 12. Как диагностировать, почему не приходят данные?
Пошагово:
1. Проверить доступность хоста (ping).
2. Проверить порт 10050/10051.
3. Проверить статус агента.
4. Проверить логи:
    - zabbix_agentd.log
    - zabbix_server.log
5. Проверить key item'а.
6. Проверить timeout.
7. Проверить тип item (active/passive).
8. Проверить firewall.
9. Посмотреть Latest data.
10. Проверить очередь: `zabbix[queue]`.

Если active agent:
- Проверить ServerActive
- hostname должен совпадать
Если proxy:
- Проверить связь proxy ↔ server.
